Kindle for Mac で開いている本を、ページを自動でめくりながら連続スクリーンショットし、
`kindle_screenshots/<日時>/001.png, 002.png …` として保存します。
内部では `capture_kindle.py`（pyautogui 利用）を実行します。

## 引数（任意・スペース区切り）
- `$1` … 撮影するページ数（省略時は 999 を上限とし、巻末で自動停止）
- `$2` … ページ送り後の待機秒数（省略時は 1.5）

入力された引数: `$ARGUMENTS`

## 実行手順
1. これから行うこと（ページ数・間隔・保存先）を 1〜2 行で要約したうえで、実行する **前に必ず**次を伝える:
   - 「カウントダウン（5秒）の間に **Kindle を最前面**にして、**本の先頭ページ**を開いてください」
   - 「実行中はマウス・キーボードがスクリプトに使われます。終わるまで他の操作はしないでください」
2. 以下のコマンドをプロジェクト直下で実行する。
   `--key auto --rewind --app --owner --stop-repeat` は**常に付ける**（理由は下記）:

   ```
   python3 capture_kindle.py --pages 999 --key auto --rewind \
     --app "Amazon Kindle" --owner "Kindle" --stop-repeat 3
   ```

   - `$1` が指定されていれば `--pages $1` に差し替える
   - `$2` が指定されていれば `--interval $2` を付ける
   - 撮影は数十分かかるので、Bash ツールの `run_in_background` で実行し、
     完了通知を待つ間に他の作業を止めない
3. 完了したら、標準出力の「保存先: …」から日時フォルダのパスを読み取り、
   `ls -1 <そのフォルダ>` で保存枚数を確認し、**保存先パスと枚数**を報告する。
4. **必ず検証する**: 先頭 `001.png` と末尾のファイルを Read で開き、
   001 が表紙・末尾が巻末になっているか目視で確かめてから報告する
   （方向を取り違えると本が逆順で撮れるが、枚数だけ見ても気づけない）。

## オプションの意味（省略しないこと）
- `--app "Amazon Kindle"` … ページ送りキーを届かせるため毎回 Kindle を最前面にする。
  撮影中に他アプリが前面に出ても撮り逃さない。
- `--owner "Kindle"` … Kindle の**ウインドウ単位**で撮影する。これが無いと
  `pyautogui.screenshot()` がメインモニタ全体を撮るため、**Kindle を別モニタに置いていると
  全く違う画面が保存される**（実際に起きた事故）。
- `--key auto` … ページ送りの向きを自動判定する。本の端では左右を試し打ちし、
  判定できないときは本文が縦書き（右綴じ＝左送り）か横書き（左綴じ＝右送り）かを
  ページを送りながら多数決で判定する。自信が持てない場合は撮影せず停止する。
- `--rewind` … 撮影前に本の先頭まで自動で戻す。途中から開いていても全ページ撮れる。
- `--stop-repeat 3` … 同じ画面が3回続いたら巻末とみなして停止し、末尾の重複を削除する。
  ページ数が分からなくても最後まで撮り切れる。

## 補足・注意
- 依存パッケージは `python3 -m pip install --user --break-system-packages pyautogui Pillow`。
  Pillow は pyscreeze が内部で使うため必須で、pyautogui だけでは入らない。
- pyautogui がマウス/キー操作・スクリーンショットを行うため、macOS の「アクセシビリティ」
  および「画面収録」権限を、ターミナル（または実行アプリ）に付与しておく必要があります。
  実行が失敗した場合は、この権限設定を案内してください。
- **既知の限界**: 端にいるときの「先頭」と「巻末」は左右の試し打ちだけでは区別できない
  （症状が鏡像になる）。この場合は本文レイアウトで綴じ方向を確かめ、それでも
  判断できなければ **撮影せず停止する**（`--key left` / `--key right` で明示すれば実行できる）。
  判定ロジックを変更したら `python3 test_detect_direction.py` で回帰テストすること。
- 撮影後に `python3 make_pdf.py <セッションフォルダ>` で PDF にまとめられる。
- 取得画像の著作権はコンテンツ提供者に帰属します。私的利用の範囲・各サービスの利用規約を守ってください。
