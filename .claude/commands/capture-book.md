Kindle for Mac で開いている本を、ページを自動でめくりながら連続スクリーンショットし、
`kindle_screenshots/<日時>/001.png, 002.png …` として保存します。
内部では `capture_kindle.py`（pyautogui 利用）を実行します。

## 引数（任意・スペース区切り）
- `$1` … 撮影するページ数（省略時はスクリプト既定値の 425）
- `$2` … ページ送り後の待機秒数（省略時は 1.5）

入力された引数: `$ARGUMENTS`

## 実行手順
1. これから行うこと（ページ数・間隔・保存先）を 1〜2 行で要約したうえで、実行する **前に必ず**次を伝える:
   - 「カウントダウン（5秒）の間に **Kindle を最前面**にして、**最初に保存したいページ**を開いてください」
   - 「実行中はマウス・キーボードがスクリプトに使われます。終わるまで他の操作はしないでください」
2. 引数に応じて以下のコマンドを組み立て、プロジェクト直下で実行する:
   - 基本: `python3 capture_kindle.py`
   - `$1` が指定されていれば `--pages $1` を付ける
   - `$2` が指定されていれば `--interval $2` を付ける
   - 例: 引数が「120 2」なら → `python3 capture_kindle.py --pages 120 --interval 2`
   - 引数が無ければ → `python3 capture_kindle.py`（既定値で実行）
3. 完了したら、標準出力の「保存先: …」から日時フォルダのパスを読み取り、
   `ls -1 <そのフォルダ>` で保存枚数を確認し、**保存先パスと枚数**を報告する。

## 補足・注意
- `ModuleNotFoundError: No module named 'pyautogui'` が出たら `pip install pyautogui` を実行してから再試行する。
- pyautogui がマウス/キー操作・スクリーンショットを行うため、macOS の「アクセシビリティ」
  および「画面収録」権限を、ターミナル（または実行アプリ）に付与しておく必要があります。
  実行が失敗した場合は、この権限設定を案内してください。
- ページ送りが逆向きの本は、スクリプトに `--key right` を付けると右送りになります。
- 取得画像の著作権はコンテンツ提供者に帰属します。私的利用の範囲・各サービスの利用規約を守ってください。
